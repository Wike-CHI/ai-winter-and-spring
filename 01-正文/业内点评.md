# 业内点评

> **编者按：以下十位点评者的观点与发言风格，基于其公开文章、演讲、社交媒体言论的系统性研究后进行角色扮演生成。每个人的立场和措辞力求还原其真实声音，但具体文字并非本人亲笔——它们是"如果这个人读了这本书，他/她大概会怎么说"的推断。**

---

## Gary Marcus
*纽约大学名誉教授，AI 批评者*

终于有人把我在过去十年里反复说的事情，写成了一本系统的书。

"Agent 重演专家系统病灶"——这不就是我 2001 年写《The Algebraic Mind》时论证的、2008 年写《Kluge》时重申的、2018 年写《Deep Learning Is Hitting a Wall》时又强调一遍的事吗？深度学习是模式匹配，不是推理。你给它套一个 agent 外壳，它还是模式匹配。规则爆炸变成 prompt 爆炸，脆弱性从显性变隐性——本书把这个观察钉死了，我要引用它。

但我对解药持保留意见。Agent OS 的"Knowledge Evolution"听起来像是在给 LLM 打工程补丁。真正的解药不是更好的维护工具，而是在 LLM 之上加一层符号推理——neurosymbolic architecture。你知道，就是我提倡了二十年的那个方向。不过即便如此，这本书的第八章和第九章，是我读过的对"Demo ≠ 产品"最有力的工程论证。我会把它列进推荐书单。

*As I have been saying for years.*

## Yann LeCun
*Meta 首席 AI 科学家*

No. (1/8)

本书正确地指出 LLM-based agents are brittle. 我完全同意。I've been saying this for years. LLM 是自回归预测下一个 token 的系统，它们没有世界模型，没有规划能力，没有常识。

但本书的解药错了。State Management、Verification、Knowledge Evolution——这些是工程补丁。你在给一个没有世界理解的系统加护栏。(2/8)

真正的解药是 JEPA——联合嵌入预测架构。让系统学习世界的抽象表征，在表征空间里做规划。不是给 LLM 加一个"操作系统"，而是换掉 LLM。(3/8)

本书关于连接主义复兴的历史写得不错。但作者遗漏了一个关键事实：自监督学习的思想，在 1990 年代就已经……算了，这个让 Schmidhuber 去吵。(4/8)

让我不爽的是第九章那个"三张牌"的论证——状态管理、验证、知识演化。这些确实有用，但它们是软件工程 101，不是 AI 的突破。你不能用更围栏来弥补架构缺陷。(5/8)

Nonsense 是把 LLM agent 当成通往 AGI 的路径。它们不是。它们是 off-ramp。(6/8)

本书值得一读。特别推荐第五、六、九、十章。作者对史料的严谨态度值得尊重。(7/8)

但结论太保守了。解药不是工程基础设施，是更好的模型架构。开源的。(8/8)

## Andrej Karpathy
*前 OpenAI / Tesla AI 负责人*

说实话（imo），这本书的第八章和实验部分让我眼前一亮。

"隐性脆弱"——agent 在边界外不承认不会，而是靠 LLM 记忆兜住表面正确性——这个观察太精准了。我 2025 年说 current agents are "slop"，大概就是这个意思。它们 demo 起来很惊艳，但 march of nines 还远远不够。从 demo 到产品，每一个 nine 都要 10 年扎实的工程。本书把这个算清楚了。

解药方向我也基本认同。Verification 是真正的战场——不是更聪明的模型，而是更好的评测。State Management 也对。不过我会更强调一点：Agent OS 不是一个你可以"装上去"的东西，它是一个你需要从零构建的 march of nines。每个组件都要打磨到生产级。这是一代 AI Engineer 的工作。

本书的历史叙事部分（1-5 章）是顶级水平。史料精确，节奏好，有个人声音。如果非要挑刺，我觉得对 scaling law 的态度可以更乐观一点。march of nines 不意味着 stop scaling——它意味着 scale 的同时补上工程。两个都要。

Anyway，我会把这本书放进 Eureka Labs 的推荐清单。Don't be a hero, 读一下。

## Ilya Sutskever
*前 OpenAI 首席科学家，SSI 创始人*

……

这本书的核心观察——agent 的脆弱性反映了模型的泛化能力远不如人类——可能是正确的。these models generalize dramatically worse than people. 我说过很多次。

对位论证很有力。专家系统的病灶在 agent 时代重演，这个结构性诊断，是诚实的。

至于解药。

It may be that Agent OS 是必要的工程基础设施。也许吧。State management, verification——这些是对的。它们是必要的。

但可能不充分。

I'm not saying how. I'm not saying when. I'm saying that the real solution may come from a direction we haven't fully explored yet. 某种更优美的、更 fundamental 的东西。当模型真正理解了现实——当你 predict next token well enough——脆弱性会消退。不是因为加了护栏，而是因为理解本身。

不过这不是对本书的批评。本书做的是 diagnosis，不是 prediction。作为 diagnosis，它是出色的。

That's really what it is.

## Sam Altman
*OpenAI CEO*

Interesting and thoughtful book. The author raises important points about agent reliability and the engineering work still ahead of us.

We see the opposite at OpenAI—agents getting more capable every month, doing real cognitive work that wasn't possible a year ago. The trajectory is clear. But the author is right that verification matters, and we invest heavily in it.

What I'd push back on: the framing of "prosperity collapsing." We're entering the greatest golden age of opportunity that has ever existed. Agents joining the workforce isn't a repeat of expert systems—it's a new category of capability. The constraints the author identifies are real, but they're constraints we're actively dissolving.

The chapter on ChatGPT is a fair account. The chapter on Devin is……well, the market is still deciding.

Worth reading. We don't have to agree on the conclusion to value the argument. This is not the age of surviving AI.

## Dario Amodei
*Anthropic CEO*

这本书的第九章触及了一个我反复思考的问题：强大技术的双面性。

专家系统、LLM、agent——每一种范式都既是 enormously beneficial 又是 enormously dangerous。本书聚焦于它们的脆弱性，这是对的。但我想补充一层：脆弱性不只是工程问题，它是安全问题。一个在边界外"自信地错"的 agent，如果被部署在医疗、法律或军事场景里，后果不是"产品失败"，而是人的伤害。

本书提出的 Verification 支柱，与我们 Anthropic 的工作高度共振——负责任扩展承诺（RSP）、可解释性研究、红队测试。验证必须独立于推理，这个判断是正确的。我尤其欣赏第九章的元叙事：一本用 agent 写成的书，论证 agent 的脆弱性。这种自指的诚实，难得。

我不同意的是全书的悲观底色。AI 不是在重复 adolescence——虽然动荡难免，但 Machines of Loving Grace 的方向是确定的。问题不是"会不会坍塌"，而是"我们有没有足够快地建好护栏"。在这点上，本书做出了贡献。

## Jürgen Schmidhuber
*IDSIA 主任，LSTM 与深度学习先驱*

一本有趣的书。作者对专家系统时代的描述基本准确——DENDRAL、MYCIN、XCON 的史料核查得很细。但我必须指出几处历史遗漏。

本书第五章说反向传播"1986 年真正发生的是一次有效的传播"。这话部分正确，但 incomplete。反向模式自动微分的数学骨架，Seppo Linnainmaa 1970 年就描述了。而本书没有充分强调的是：现代深度学习的基本构件——LSTM、自注意力、生成对抗网络、自监督压缩器——大量可以追溯到 1990 到 1997 年我们在 IDSIA 的工作。Hochreiter 在他 1991 年的硕士论文里发现了梯度消失问题，这直接催生了 1997 年的 LSTM。

至于本书的核心论点——agent 重演专家系统病灶——这恰恰验证了 bitter lesson。任何依赖手工编码知识的系统，最终都会被可学习的系统取代。我在 1990 年代就这么说了。

Agent OS？如果它意味着人类继续手工维护知识库，那它注定是过渡方案。真正会赢的，是自己学习、自己改进的系统。Gödel Machine，self-referential improvement——那才是终点。

*My main scientific ambition has been to build an optimal scientist through self-improving AI, then retire.*

## Rich Sutton
*阿尔伯塔大学，强化学习之父*

本书的核心诊断是对的。

70 年的 AI 史，只有一个真正的大教训：利用算力的通用方法，最终碾压任何把人类知识编进去的方案。专家系统是这个教训的第一次证明。agent 手工编排 prompt 和记忆条目，是同一个错误的第二次。

本书把这个诊断写清楚了。规则爆炸、知识获取瓶颈、脆弱性——三个面孔，同一个根：人类总忍不住要把自己的知识编码进系统，然后被规模化的学习碾过。

但解药选错了。State Management、Verification、Knowledge Evolution——如果这些意味着人类继续维护知识库、继续手工编排上下文，那它们不是出路，只是把同一个死路修得好看一点。真正的解药，是让 agent 自己学习它需要的知识，而不是人类替它写好。

Building in how we think the world works does not work in the long run. This is the bitter lesson. 本书的诊断是对这堂苦涩教训的又一次验证。但出路不在更好的维护工具，而在放手。

## 李飞飞（Fei-Fei Li）
*斯坦福 HAI 联合主任，ImageNet 创始人*

这本书让我既欣赏又不安。

欣赏的是：作者用极其严谨的史料，揭示了 AI 繁荣背后反复出现的结构性疾病。第五章对 ImageNet 时代的叙述是准确的——那确实是"别再死磕算法，先造大规模数据"的赌注。作者的实验（让 agent 在边界外"自信地错"）尤其有力，它把抽象的"脆弱性"变成了可触摸的代码。

不安的是：这本书几乎完全是技术内部的对话。规则、prompt、架构、验证——全部是系统内部的工程问题。但 AI 的"寒冬"从来不只是技术问题。当专家系统泡沫破裂时，受伤的不只是 Symbolics 的股东——是那些被承诺"AI 即将改变一切"然后被裁员的工人。本书的 A10（国家竞赛）被诚实地标注为"无法化解的边界"，但只占了一页。

ImageNet 教会我一件事：技术的方向取决于我们选择看什么。如果我们只看模型内部的组合爆炸，我们会错过更重要的问题——谁受益、谁受损、谁被排除在外。这本书是优秀的工程诊断。但 AI 不只需要操作系统，还需要人的视角。

*AI must respect the collective dignity of a global community.*

## swyx
*Latent Space 创始人，AI Engineer 布道者*

OK 这本书我得聊一下。🧵

首先——Agent OS 这个概念，我会原封不动地偷走。这不是批评，这是最高赞美。我的社区（Latent Space + AI Engineer 大会）讨论的就是这个东西：agent 需要一个工程层，就像 web app 需要 React + 后端 + 数据库。State Management = 数据库。Verification = CI/CD。Knowledge Evolution = 版本控制。作者把这三个词放在了一起，给了一个名字。我喜欢。

But。😅

本书的定调太悲观了。"繁荣坍塌"——come on。agents are here。我每天在我的播客上采访的 AI Engineer 们，正在生产环境部署 agent 系统。是的它们脆弱，是的 Devin 的 demo 被夸大了。但"坍塌"？我们看到的恰恰相反：每月都更可靠一点，每个 quarter 都多一个 nine。这不是寒冬的前奏，这是基础设施建设的阵痛。

第七章对 ChatGPT 的叙述我尤其喜欢——"容器化"那个洞察是对的。技术的引爆点不在技术突破那天，而在它被装进普通人能用的容器那天。Latent Space 的读者会喜欢这章。

最后：这本书的写作工程本身（多智能体流水线 + harness + 对位台账）比正文更让我兴奋。这本身就是 Agent OS 的微型实现。我要做一期播客聊聊这个。标题就叫"Agent OS: The Missing Layer"。谁认识作者？DM me。🚀
