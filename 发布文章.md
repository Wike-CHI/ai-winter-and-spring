# 我用多智能体写完了一本 AI 技术史：专家系统、Agent 与四十年后的重演

> 这不是一篇"我用 AI 写了一本书"的帖子。那个说法已经烂了。
>
> 我要做的是：把写作拆成六个角色的流水线，加上史料核证、对位台账、回归测试和可运行实验，然后把全部产物开源。它论证的是——Agent 时代正在重演专家系统的病灶，解药不是更大的模型，而是 Agent Operating System。

---

## 这本书讲了什么

书名叫《智能之冬与智能之春：专家系统、互联网、大模型与 Agent 的四十年战争》。

核心论点一句话：

> 每一次 AI 繁荣都坍塌于同一个病灶——当系统规模增长时，"知识"的维护成本随规模指数级爆炸。四十年前发作于规则，今天发作于 prompt。

全书十章，三个幕：

- **第一幕（符号主义兴衰）**：1956 达特茅斯 → 专家系统狂热 → 规则爆炸 → 1987 寒冬
- **第二幕（连接主义复活）**：反向传播 → 统计学习 → AlexNet → Transformer → ChatGPT
- **第三幕（Agent 时代）**：Devin/SWE-bench 狂热 → 历史重演？→ Agent Operating System

第三章和第九章是镜像对位章——专家系统的每一个病灶，在 Agent 时代找到了精确的当代对应物：

| 专家系统时代 | Agent 时代 |
|---|---|
| Rule Explosion（XCON 17,500 条规则） | Prompt Explosion |
| 知识获取瓶颈 | 上下文工程瓶颈 |
| Brittleness（边界外显性崩溃） | 隐性脆弱（靠记忆伪装推理，更危险） |
| Knowledge Engineer 新职业 | Prompt/Agent Engineer |
| Lisp Machine 泡沫 | Agent 框架泡沫 |
| XCON"一年省四千万"Demo | Devin"自主写完 App"Demo |

十项，全部命中。

## 它不是"AI 辅助写作"

我不想把这包装成"我用 ChatGPT 写了一本书"。那句话没有信息量。

这本书的写作过程，是一套**多智能体工程流水线**。每一章都经过六个角色：

```
Researcher（史料核证）──► Writer（起草）──► Fact-Checker（逐句复核）
                                       └──► Continuity Editor（对位一致性）
                                       └──► Style Editor（去 AI 腔）
                  Orchestrator（编排）全程调度
```

Researcher 并行联网核证每个人名、年份、数字、引语原文。Fact-Checker 不信任 Researcher 的初判，用一手来源独立交叉复核。Continuity Editor 检查第三章埋的伏笔是否在第九章精确呼应。Style Editor 扫除 AI 腔（三段排比堆砌、形容词通胀、空洞升华）。

这不是"给模型一个 prompt 然后等它输出"。这是带质检闸门的生产系统。

## 它纠正了 50 多处史实错误

Fact-Checker 层不是摆设。它在核证过程中揪出了 50 多处史实问题——有些是大纲里就有的，有些是二手资料里广泛流传的：

- **第五代机预算"8.5 亿美元"是换算错误**——按 1982 年汇率实为约 2.3-4 亿美元
- **Feigenbaum 1977 的原句被改写拼接**——通读一手 PDF 后改用逐字原文
- **GPT-1 早于 BERT 四个月**——大多数资料搞反了这个时序
- **Devin 的 SWE-bench 13.86% 从未被官方排行榜独立登载**
- **LangChain 不是 2017 年创立的，是 2022 年 10 月开源**

每一条都有可点开核实的一手来源 URL，附在每章的史料核证附录里。

## 它有自己的实验证据

我不只是转述文献。我亲手跑了一个 Agent 边界实验。

用 langchain 1.1 + langgraph 1.0 + GLM-5.2，我构建了两个 agent：一个通用 agent（带计算器、元素查询等通用工具），一个 DENDRAL 升级版专用 agent（带质谱解析、官能团库等化学专用工具）。

三个发现：

1. **Prompt 爆炸**：通用 agent 的 prompt 从 30 token 涨到 160 token 时，工具调用从 2 次暴涨到 8 次；专用 agent 因知识固化在工具里，恒定 4 次。
2. **长程漂移**：任务从 3 步延长到 8 步，通用 agent 消息数膨胀；专用 agent 恒定。
3. **隐性脆弱**：把一个工具库不覆盖的金属有机化合物丢给 agent，它没有承认超范围，而是靠 LLM 记忆给出了看起来正确的答案——但绕过了工具校验层。

第三点是最锋利的发现。1985 年的专家系统边界外会**显性崩溃**（给荒谬答案），容易被发现。2026 年的 Agent 边界外会**隐性失效**——靠记忆伪装推理，看起来对了但过程是假的。后者更危险。

写完全书后，我建了一套 harness 工程，把全部编排资产解析为 Python 对象，然后写了 36 项自动化回归测试：

## 还有一个可交互的复现脚本

你甚至可以亲手体验规则爆炸：

```bash
cd 06-复现实验
python reproduction.py
```

这个脚本用纯 Python 实现了一个简化版 MYCIN（感染诊断专家系统）。你亲手加规则，亲眼看到冲突从 1 个暴涨到 68 个（O(N²) 组合爆炸），亲手触发脆弱性（给系统一个边界外的输入，它会 confidently 开错药）。然后快进到 Agent 时代，同样的知识塞进 prompt vs 固化成工具——4.4 倍 token 差异。

四十年压缩进一个命令行。

## 所以，这本书到底是什么？

最准确的定位是：**一部用多智能体工程方法完成的 AI 技术史论长文 / 开放式研究型小书。**

它不是最终出版版。它是开源的，欢迎 Issue 挑错。

但它有一个我觉得最有价值的闭环：

> **我用 Agent OS 的方法（状态管理、知识演化、独立验证），写了一本论证 Agent OS 必要性的书。写作过程本身，就是论点的实验场。**

这本书在写作中遭遇的每一次上下文坍缩、每一次对位漂移、每一次 Fact-Checker 拦截的幻觉，都是书中所诊断的病灶的一次真实发作。而它最终能交付到你手上——不是因为 Agent 可靠，而是因为围绕它建造了工程基础设施。
