## 一、术语表用途与使用方法

本书横跨符号主义、连接主义、Agent 三个时代，每个时代都有自己的黑话。如果不统一，会出现三类问题：

1. **同人异译**：第三章写"知识工程师"，第九章写"提示词工程师"，对位感就散了。
2. **异人同译**：把 Herbert Simon 和 Paul Samuelson 都叫"西蒙"，读者混淆。
3. **AI 腔渗入**：术语是最容易藏 AI 腔的地方（"赋能""抓手""范式跃迁"）。

本表是全书词汇的**单一事实来源**。

### 字段说明

| 字段 | 说明 |
|---|---|
| **中文统一译法** | 全书正文必须用此译法 |
| **英文/原文** | 首次出现时括注 |
| **禁用译法** | Style Editor 扫除的对象 |
| **首次出现规则** | 如何引入该术语（括注英文 / 注人名等） |
| **备注** | 溯源、争议、注意事项 |

### 使用纪律

1. **Writer 起草前**：通读本表，锁定译法。
2. **首次出现**：按"首次出现规则"引入；后续一律用中文，不再重复英文。
3. **Style Editor 定稿前**：用本表"禁用译法"列做全文检索，发现即改。
4. **新增术语**：每章若引入新术语，Writer 须先回填本表，再用于正文——**术语先入库，再入文**。
5. **人名原则**：有公认中文名的用中文名（首次注英文原名）；无公认中文名的用英文原名，不硬译。

## 四、易混术语辨析（写作雷区）

这些是本书最容易写错、最需要刻意区分的概念对：

| 易混对 | 区分 | 写作要求 |
|---|---|---|
| **核证**（fact-check）vs **验证**（verification） | 核证=史实是否属实（本书 Fact-Checker 角色）；验证=Agent 产出是否正确（第十章 Verification 支柱） | 全书严格区分，不混用 |
| **规则爆炸**（Rule Explosion）vs **Prompt 爆炸**（Prompt Explosion） | 前者专家系统，后者 Agent 时代，是**对位 A1** | 对位出现时务必成对，凸显"同一个病灶换衣服" |
| **脆弱性**（brittleness）vs **幻觉**（hallucination） | 前者专家系统边界外崩溃，后者 LLM 编造内容；**对位 A3/A4** | 不要混为一谈，但要点出同构性 |
| **知识获取瓶颈** vs **上下文工程瓶颈** | 对位 A2 | 成对出现 |
| **连接主义** vs **符号主义** | 两大阵营；本书"三方"还有 Agent 阵营 | 第一次提及时给简明定义 |
| **workflow** vs **agent** | Anthropic 的关键区分：确定性管道 vs 模型自主决策 | 第八章必须讲清，是第十章的伏笔 |
| **Demo** vs **产品** | 对位 A9 的核心张力 | "Demo ≠ 产品"是反复出现的母题 |

## 五、文风禁用词（AI 腔黑名单 · 与术语表联动）

> 以下非术语，但属于"用词规范"，Style Editor 按此扫除。列于此处便于统一检索。

**一律禁用**：
- 口水连接词：值得一提的是、综上所述、众所周知、不言而喻、随着……的发展、在当今……背景下、由此可见
- 形容词通胀：革命性的、颠覆性的、划时代的、前所未有的（除非引语）、无与伦比
- 三段排比堆砌：不仅……而且……更是……
- 空泛总结：历史总是重演、未来已来、AI 将改变一切

**慎用**（需具体支撑）：
- 大量、许多、一段时间后、近年来 → 必须换成具体数字/年份
- 业界认为、专家表示 → 必须给出具体人或机构
