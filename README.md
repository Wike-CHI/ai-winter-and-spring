## 这本书在讲什么

每一次 AI 繁荣都坍塌于同一个病灶：当系统规模增长时，"知识"（规则 / 特征 / 上下文）的维护成本随规模指数级爆炸。每一次复兴，都源于发明了一种能让知识以亚线性成本被复用的"新容器"。

**专家系统死于 Rule Explosion，Agent 正在同一条轨道上重演 Prompt Explosion 与 Context Collapse——唯有把易腐的"上下文"固化为可演化的工程资产（即 Agent Operating System），AI 才可能第一次逃出"繁荣—寒冬"的四十年周期。**

本书贯穿五条叙事主线：

```
1956 达特茅斯 ─► 1987 第一个 AI 之冬 ─► 2017 Transformer ─► 2022 ChatGPT ─► 2024–2026 Agent 狂热
```

## 这本书的特别之处：用多智能体写一本论证多智能体的书

本书不是"用 AI 辅助写作"。它是一个**多智能体协作工程**——每一章都由一套严格的写作流水线产出：

```
Researcher（史料核证）──► Writer（起草）──► Fact-Checker（逐句复核）
                                       └──► Continuity Editor（对位一致性）
                                       └──► Style Editor（去 AI 腔）
                  Orchestrator（编排）全程调度
```

每一章的每一个史实断言，都有独立的 Fact-Checker 用一手来源交叉验证。截至第六章，这套流水线已经**揪出并修正了 40+ 处史实问题**——包括"第五代机预算换算错误（8.5 亿美元→实为约 4 亿）"、"Feigenbaum 1977 原句是改写拼接"、"GPT-1 早于 BERT 四个月"等硬伤。

讽刺且恰当：**我们在用第十章的 Agent OS 思想（Project Memory、Verification、Knowledge Evolution），来写一本论证第十章的书。** 详见 [`03-写作编排/`](./03-写作编排/)。

## 文风

定调：**史诗感 × 工程严谨 × 批判性反思。** 不歌颂、不唱衰，像一个真正在战壕里待过的工程师兼史学家在复盘。模仿《硅谷之火》（*Fire in the Valley*）的叙事非虚构范式——具体场景开篇、数字与年份锚定每个判断、反对空泛赞美与廉价犬儒。

## 核心对位（全书思想骨架）

第三章（专家系统的病灶）与第九章（Agent 时代的重演）是严格镜像：

| 专家系统时代 | Agent 时代 | 共同病灶 |
|---|---|---|
| Rule Explosion 规则爆炸 | Prompt Explosion | 显式知识的组合爆炸 |
| 知识获取瓶颈 | 上下文工程瓶颈 | 隐性知识难以显式化 |
| Brittleness 脆弱性 | 长程任务漂移 | 边界外灾难性失效 |
| Knowledge Engineer | Prompt/Agent Engineer | 手工编排知识的工匠 |

完整的 10 条对位表见 [`03-写作编排/对位台账.md`](./03-写作编排/对位台账.md)。
