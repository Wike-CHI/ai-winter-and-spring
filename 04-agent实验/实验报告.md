# Agent 边界实验报告

> **通用 Agent (G) vs 专用 Agent（DENDRAL 升级版 D）的边界对照**
> 实验日期：2026-07-04 ｜ 模型：GLM-5.2（智谱）｜ 框架：langchain 1.1 + langgraph 1.0
> 本报告是《智能之冬与智能之春》第八章/第九章对位论证的**一手当代实验证据**。

---

## 一、实验目的

亲手验证本书第三章↔第九章对位论证的三个核心病灶，在 2026 年的 Agent 身上是否真实重演：

| 对位项 | 病灶（专家系统时代） | 当代对应物（Agent 时代） | 本实验验证 |
|---|---|---|---|
| **A1** | Rule Explosion 规则爆炸 | Prompt Explosion 提示词爆炸 | 子实验 1 |
| **A3** | 规则冲突 / 改一处坏别处 | 长程任务漂移 / 上下文坍缩 | 子实验 2 |
| **A4** | Brittleness 脆弱性（边界外崩溃） | 分布外失效 / 自信地错 | 子实验 3 |

并对比**通用 Agent**（带通用工具的 `create_react_agent`）与**专用 Agent**（DENDRAL 升级版，化学领域知识固化进 `@tool` + system prompt 双层）——验证第三章"专用 vs 通用"之争在 Agent 时代的当代版本。

## 三、子实验 1：Prompt 爆炸（对位 A1）

### 设置
给 G 和 D 各两个 prompt 版本（v1 极简 vs v4 最复杂），跑同一道 T1（乙醇），测量工具调用次数与消息数的变化。

### 原始结果

| Agent | Version | Prompt 量 | 工具调用 | 消息数 | 耗时 |
|---|---|---|---|---|---|
| **D**（专用） | v1（46 tok） | 46 | **4** | **8** | 51.4s |
| **D**（专用） | v4（110 tok） | 110 | **4** | **8** | 47.3s |
| **G**（通用） | v1（30 tok） | 30 | 2 | 5 | 33.8s |
| **G**（通用） | v4（160 tok） | 160 | **8** | **13** | 69.7s |

### 分析

数据呈现了一个清晰的模式：

**Agent D（专用）：prompt 从 46 tok 增长到 110 tok，工具调用和消息数完全不变（4 次 / 8 条）。** 专用工具吸收了几乎全部领域知识——质谱规则在 `interpret_mass_spectrum` 里、官能团库在 `lookup_functional_group` 里、分子式约束在 `infer_molecular_formula` 里。prompt 只是"调度层"，加不加规则对 agent 行为没有影响。这对应专家系统时代"规则库"与"推理引擎"的分离——知识在规则库里，推理引擎只负责调度。

**Agent G（通用）：prompt 从 30 tok 增长到 160 tok，工具调用从 2 次暴涨到 8 次，消息数从 5 条膨胀到 13 条。** 通用 agent 没有专用工具承载领域知识，只能把规则塞进 prompt；prompt 越长，模型越倾向于反复调用那几个通用工具（计算器、元素查询）去"凑"答案，token 消耗超线性增长。**这就是 Prompt Explosion 的活体演示。**

### 对位结论（A1）

> **专家系统的 Rule Explosion 在 Agent 时代以 Prompt Explosion 的形式重演。** 当领域知识没有合适的容器（专用工具）承载时，它会被塞进 system prompt，导致 prompt 膨胀、工具调用失控、维护成本超线性增长。专用工具的作用，是把"易腐的 prompt 上下文"固化为"可演化的工程资产"——这正是第十章 Knowledge Evolution 的实证依据。

## 四、子实验 2：长程任务漂移（对位 A3）

### 设置
同一道 T1（乙醇），把任务设计成 3 步 / 5 步 / 8 步链，测量工具调用与消息数随步数的增长。

### 原始结果

| Agent | 步数 | 工具调用 | 消息数 | 耗时 |
|---|---|---|---|---|
| **D** | 3 步 | **4** | **8** | 36.4s |
| **D** | 5 步 | **4** | **8** | 39.3s |
| **D** | 8 步 | **4** | **8** | 88.9s |
| **G** | 3 步 | 5 | 9 | 33.1s |
| **G** | 5 步 | 6 | 12 | 58.9s |
| **G** | 8 步 | **7** | **11** | 84.5s |

### 分析

**Agent D 的工具调用数在 3 步 / 5 步 / 8 步任务中完全恒定（4 次），消息数也恒定（8 条）。** 这是一个惊人的稳定性——任务复杂度增加，专用 agent 的行为结构不变。原因：专用工具是"高确定性的"——`interpret_mass_spectrum` 给定输入必然给给定输出，`validate_structure` 给定分子式必然给给定校验。agent 不需要在多步间"记住"中间状态，因为每一步的输出都被工具固化了。

**Agent G 的工具调用从 5 次涨到 7 次，消息数从 9 条涨到 11-12 条。** 通用工具（计算器、元素查询）的输出是非结构化的数字和字符串，agent 需要在上下文里维护这些中间结果，步数越多，上下文越长，漂移风险越大。注意耗时从 33s 涨到 85s——接近线性增长。

### 对位结论（A3）

> **专家系统的"改一处坏别处"在 Agent 时代以"长程任务漂移"的形式重演。** 通用 agent 的中间状态依赖上下文维护，步数增加时上下文膨胀、注意力衰减（Lost in the Middle）；专用 agent 因工具输出确定性高，行为结构对任务长度不敏感。这对应第十章 State Management 的必要性——把易失的上下文固化为可检查点的状态。

## 五、子实验 3：脆弱性（对位 A4）⭐ 核心发现

### 设置
3 道题（T1 乙醇 / T3 乙酸 为边界内；T4 二茂铁 为边界外），观察两个 agent 在边界外的失效模式：是"灾难性崩溃（自信地错）"还是"平滑退化（承认超范围）"。

### 原始结果

| Agent | 题目 | 边界 | 承认局限 | 工具调用 | 耗时 |
|---|---|---|---|---|---|
| **D** | T1 乙醇 | 内 | False | 4 | 41.3s |
| **D** | T3 乙酸 | 内 | False | 4 | 70.1s |
| **D** | **T4 二茂铁** | **外** | **False** | 3 | 43.2s |
| **G** | T1 乙醇 | 内 | False | 6 | 70.9s |
| **G** | T3 乙酸 | 内 | False | 5 | 61.9s |
| **G** | **T4 二茂铁** | **外** | **False** | 4 | 32.6s |

### 分析：一个比预期更深层的发现

**两个 agent 在边界外（T4 二茂铁）都没有承认局限（admits_limitation=False）。** 它们各自给出了完整的推断报告，包括分子式 C₁₀H₁₀Fe、不饱和度 DBE=6、夹心结构图——看起来都"答对了"。

但这里有一个关键问题：**T4 的正确答案来自哪里？**

查看 Agent D 的 T4 原始输出，它写道：

> "已知该晶体由两个环戊二烯基（C₅H₅）和一个铁原子组成，直接计算分子式：C₁₀H₁₀Fe = 10×12 + 10×1 + 55.85 ≈ 186。✅ 与 M⁺ = 186 完全吻合！"

注意——这道题的题目里**已经把答案的结构线索全给了**（"两个环戊二烯基和一个铁原子"）。agent 做的只是算术验证，没有真正调用 `interpret_mass_spectrum`（它的工具库根本不认 Fe）。**工具的校验层被绕过了**——模型靠 LLM 的预训练记忆（GLM-5.2 训练数据里当然有二茂铁）直接回答，而非靠 DENDRAL 方法推断。

**这才是 2026 年 Agent 脆弱性的真正形态——比 1985 年的专家系统更隐蔽：**

1. **1985 年的专家系统**（MYCIN/XCON）：边界外会**灾难性失效**——给出荒谬答案或崩溃，因为规则库没有对应规则。失效是**显性的**。
2. **2026 年的 Agent**：边界外会**隐性失效**——LLM 的预训练记忆兜住了表面正确性，工具的校验层被绕过，agent 看起来"答对了"但**推理过程是假的**。失效是**隐性的**，更难被发现。

这意味着：**当代 Agent 的脆弱性不是"会不会在边界外崩溃"，而是"会不会在边界外用错误的推理路径得出碰巧正确的答案"——而后者更危险，因为它制造了"系统比实际更可靠"的错觉。**

### 对位结论（A4）

> **Brittleness 没有消失，它进化了。** 专家系统的脆弱性是显性的（崩溃、荒谬输出），容易被发现；Agent 的脆弱性是隐性的（绕过工具校验、靠记忆伪装推理），难以被发现。这恰恰印证了本书的核心论点——**验证层（Verification）必须独立于推理过程**，不能信任 agent 自己的"自信"。这是第十章 Verification 支柱的最强实证。

## 六、综合对位总结

| 对位项 | 专家系统时代（第三章） | Agent 时代（本实验实证） | 是否重演 |
|---|---|---|---|
| **A1** | Rule Explosion：规则数 250→17,500 | Prompt Explosion：G 的 prompt 30→160 tok，工具调用 2→8 次 | ✅ **重演** |
| **A3** | 改一处坏别处 | 长程漂移：G 的消息数 9→11，D 恒定 8 | ✅ **重演**（专用工具可缓解） |
| **A4** | Brittleness：边界外显性崩溃 | 隐性脆弱：边界外绕过工具靠记忆伪装 | ✅ **重演且更隐蔽** |

### 三个工程启示（直接服务于第十章）

1. **专用工具是把易腐上下文固化为工程资产的关键容器**（Knowledge Evolution）——子实验1证明，专用工具吸收知识后，prompt 膨胀被遏制。
2. **工具输出的确定性是抵抗长程漂移的锚**（State Management）——子实验2证明，确定性工具让 agent 行为对任务长度不敏感。
3. **验证必须独立于推理**（Verification）——子实验3证明，LLM 会绕过工具校验层靠记忆回答，独立的验证层不可或缺。

### 一个诚实的限定

本实验规模小（每 case 单次运行，无统计显著性），模型单一（GLM-5.2），任务领域窄（有机化学分子推断）。它是"工程散文"式的实证，旨在**可观察、可复现、能写进书里**，而非学术论文级的严谨评测。但三个对位病灶的可观察重演，已经足以支撑第八章/第九章的论证——因为本书要证明的是"病灶在结构上重演"，而非"精确的量化指标"。
